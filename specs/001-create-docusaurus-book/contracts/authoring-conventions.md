# Authoring Conventions & Contracts

**Date**: 2025-12-07
**Context**: This document defines the "contracts" for content authors. In this file-based project, the contracts are the conventions and data structures an author must use to ensure their content is correctly processed and displayed by Docusaurus.

## 1. Markdown Page Contract (`.md`)

Every page (a `.md` or `.mdx` file) can contain front matter metadata at the top.

### Front Matter

```yaml
---
id: unique-page-id
title: "Page Title"
sidebar_label: "Short Label for Sidebar"
---

# Page Content Starts Here...
```

- **`id`** (optional): A unique identifier for the page. If not provided, the filename is used as the ID.
- **`title`** (optional): The main title of the page, which will be used as the `<h1>` and the browser tab title.
- **`sidebar_label`** (optional): The text that will appear in the navigation sidebar. If not provided, the `title` is used.

## 2. Sidebar Configuration Contract (`sidebars.js`)

The `sidebars.js` file defines the structure of the navigation sidebar. It is the primary mechanism for organizing chapters and pages.

### Example `sidebars.js`

```javascript
/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  // By default, Docusaurus generates a sidebar from the docs folder structure
  tutorialSidebar: [{type: 'autogenerated', dirName: '.'}],

  // But we can create a sidebar manually
  // tutorialSidebar: [
  //   'intro',
  //   {
  //     type: 'category',
  //     label: 'Chapter 1: The Basics',
  //     items: ['chapter-1/page-1', 'chapter-1/page-2'],
  //   },
  //   {
  //     type: 'category',
  //     label: 'Chapter 2: Advanced Topics',
  //     items: ['chapter-2/overview'],
  //   },
  // ],
};

module.exports = sidebars;
```

- **`type: 'autogenerated'`**: This is the simplest approach. Docusaurus will create the sidebar based on the folder structure in `docs`. This is the recommended approach for this project to keep things simple.
- **`type: 'category'`**: This represents a "Chapter". It's an expandable section in the sidebar.
    - **`label`**: The title of the chapter.
    - **`items`**: An array of page `id`s that belong to that chapter.

By adhering to these conventions, authors can ensure their content is correctly integrated into the digital book.
